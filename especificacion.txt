Especificacion del producto

1. Descripcion del producto
Aplicacion web de compra y venta de articulos.

2. Objetivo
Permitir a los usuarios:
- Registrarse
- Iniciar sesion
- Publicar articulos en venta
- Ver publicaciones disponibles
- Agregar productos al carrito
- Guardar publicaciones en favoritos
- Simular una compra

3. Flujo principal de uso
1. El usuario ingresa a la aplicacion mediante la URL local.
2. Si ya tiene cuenta, puede iniciar sesion.
3. Si no tiene cuenta, debe poder acceder al registro.
4. Una vez autenticado o registrado, accede a la pagina principal con su sesion iniciada.
5. Si el usuario no esta logueado, solo deben mostrarse las opciones `Iniciar sesion` y `Registro`.

4. Funciones principales

4.1. Registro e inicio de sesion
- El sistema debe permitir registrar usuarios con email y contrasena.
- El boton `Iniciar sesion` debe estar orientado a usuarios ya registrados.
- Si el usuario no esta registrado, debe existir acceso claro a la opcion `Registro`.
- El boton `Cerrar sesion` debe finalizar la sesion y redirigir al login.

4.2. Pagina principal
- Debe mostrar al usuario autenticado el menu principal.
- Cada seccion debe incluir un boton `Volver al menu principal`.

4.3. Publicaciones de venta
- El usuario debe poder crear publicaciones de venta.
- Cada publicacion debe permitir ingresar:
  - Articulo
  - Precio
  - Descripcion
  - Foto por URL (solo una)
  - Estado de la publicacion
  - Stock
- En la seccion de publicaciones se debe poder:
  - Ver el listado de publicaciones en estado publicado
  - Ver el detalle de cada publicacion
  - Agregar productos al carrito
  - Eliminar productos del carrito
  - Simular una compra o pago
  - Ordenar publicaciones por precio de menor a mayor
  - Ordenar publicaciones por precio de mayor a menor
  - Ordenar publicaciones por mas recientes

4.4. Gestion de mis publicaciones
- En la pantalla de gestión de mis publicaciones debe existir la opcion de:
  - Editar publicaciones
  - Eliminar publicaciones
- Cada publicacion debe manejar los siguientes estados:
  - Borrador
  - Publicado
  - Vendido
- Cuando el stock llegue a cero, el estado debe cambiar automaticamente a `Vendido`.
- mostrar todos los campos de la publicación

4.5. Favoritos
- Debe existir una pestana `Favoritos`.
- En esa seccion se deben mostrar las publicaciones guardadas por el usuario.

5. Navegacion
- Debe existir un boton para volver a la pagina principal o menu principal desde cada seccion.
- En la vista de creacion de publicaciones tambien debe existir un boton `Volver a pagina principal`.
