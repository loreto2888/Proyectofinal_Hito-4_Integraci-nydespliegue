# Pendientes de implementacion

## Prioridad alta
- Implementar carrito de compras en frontend y backend.
- Permitir agregar productos al carrito desde el listado y desde el detalle de publicaciones.
- Permitir eliminar productos del carrito.
- Implementar flujo de simulacion de compra o pago.
- Corregir el mapeo de datos entre backend y frontend para publicaciones.
- Ajustar `PostCard` para usar los campos reales devueltos por la API.
- Ajustar `PostDetail` para usar los campos reales devueltos por la API.
- Corregir `Profile` para listar correctamente las publicaciones del usuario autenticado.

## Prioridad media
- Agregar formulario de registro con campo de nombre explicito.
- Redirigir `Cerrar sesion` al login en lugar de la pagina de inicio.
- Redirigir despues de login y registro al menu principal segun la especificacion.
- Agregar boton `Volver al menu principal` en todas las secciones requeridas.
- Hacer que el boton de volver en `NewPost` vaya explicitamente a la pagina principal.
- Implementar edicion de publicaciones desde el frontend usando el endpoint `PUT /api/posts/:id`.
- Implementar eliminacion de publicaciones desde el frontend usando el endpoint `DELETE /api/posts/:id`.
- Conectar favoritos a la interfaz para marcar y desmarcar publicaciones.

## Prioridad baja
- Revisar si el campo `Titulo` debe renombrarse visualmente a `Articulo` para alinearse con la especificacion.
- Evaluar si `status`, `category` y `location` deben poder editarse desde la interfaz en vez de quedar fijos.
- Mejorar validaciones y mensajes de error en login, registro y creacion de publicaciones.
- Agregar pruebas para los flujos pendientes del frontend y para endpoints faltantes del backend.

## Observaciones tecnicas
- El backend retorna `mainImage`, `user` e `isFavorite`, pero el frontend hoy usa `imageUrl`, `author` y `liked`.
- El detalle de publicacion puede quedar incompleto porque reutiliza datos resumidos del contexto en lugar de cargar siempre el detalle completo.
- El backend ya soporta favoritos, edicion y eliminacion de publicaciones, pero esas capacidades no estan expuestas completamente en la UI.
